大概总共花了两天多的时间，搭建了这个Blog。对于计算机小白的我，即使现在看来是按图索骥的事情，事实上也并不容易。咨询先辈、查找教程，从配置环境到现在写下第一篇博文，我也完成了这么一个小小的学习到应用的过程。

为什么我要搭建这个Blog?

下载文件、查找教程、修改代码、配置环境、学习语言……有人问：你不觉得麻烦吗？我的确觉得麻烦。比起其他诸如豆瓣、微信等同样可以让你轻轻松松用文字记录生活的平台，去搭建一个Blog，然后在VScode里用MarkDown语法码字似乎显得多此一举。

但是这就是我现在正在做的事情。

之前咨询过先辈，得到的回答大概是：hexo相当方便而且实用，在这个自己搭建的网站里可以不受限制的完全自治，你还可以自定义自己的域名，安装各种有意思的插件等等，而MarkDown语法不仅学起来快，还是装x利器，用户体验相当不错。

就好像一个摄影爱好者在向我介绍iPhone11的新广角有多么意义非凡，而我又对摄影一无所知一样，不停的点头，发出几声赞叹的声音，然后听得个一知半解。

而现在我在这里写下这些文字的时候，对于自己搭建Blog的意义，我的理解已经发生了改变。那更多的是一种姿态。一个处在互联网和计算机科技高速发展时代的求学者，某种程度上应有的姿态。

Blog里都写些什么？

我这人，相当怠惰。

写文字这方面总喜欢胡思乱想，下笔（开电脑）的时候却总是能找到各种借口拖延。上了高中后，我深切的感觉到自己的思维失去了某种源泉，不知道是读书太少，还是应试作文教育让我实在是有些迷惘，某年某月的某一天，自己就那么枯竭了。

上大学后，写字没有了作文要求的束缚，自己便开始杂七杂八地看书和写文章。尝试着各种各样的文体的文字风格，创造的过程中虽然留下了不少残次品，但是仍然让我找到了高中时写文章所失却的某种快感。我想，那可能就是一种表达自我的快感吧。

Do I think? Does a submarine swim? ——Edsger Dijkstra

思考，表达。这就是我单纯的愿望。

尾声

最后，向还在看这篇Blog的可爱的你，表达我的感谢之情。人因为各种各样的原因在某处相遇，又因为各种各样的原因分别。能够认识你，我的朋友，我感到十分幸运。

谢谢。

缺失模块。
1、请确保node版本大于6.2
2、在博客根目录（注意不是archer根目录）执行以下命令：
npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: true
    tags: true
